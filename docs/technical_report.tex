\begin{abstract}
We study graph neural networks for drug safety prediction. The proposal asked whether GNNs outperform classical fingerprint baselines on a pharmaceutical task. We narrowed the application from drug-target binding to drug toxicity, built a five-level baseline ladder, and ran four GNN architectures in two configurations across six toxicity datasets and twelve Tox21 endpoints. GIN is the strongest GNN, and after a v2 upgrade (richer atom features, higher capacity, class-weighted loss) it matches fingerprints on the small or severely imbalanced classification tasks but trails on the balanced ones and on LD50 regression. Loss-side imbalance fixes inside the GNN training outperform data-side fixes (rejection sampling, SMOTE) on the classical baselines. The final phase hardens the GNN (bond-level edge features, scaffold splits with multiple seeds, focal loss for the one architecture that regressed under aggressive class weights) and takes the hardened GIN drug encoder to drug-target interaction on DAVIS, the regime where the protein partner cannot be Morgan-fingerprinted. A 3D-geometric GNN on LD50 (SchNet, Level 4) is a stretch goal.
\end{abstract}

\section{Introduction}

\subsection{Research question}

Drug development is dominated by attrition. Most candidate molecules fail in clinical trials, and a substantial fraction of those failures are toxicity-driven. Predicting safety from molecular structure alone has direct economic and ethical value, and the same modelling tools transfer to drug-target interaction (DTI), where the original proposal aimed.

The proposal asked: \emph{do graph-based and geometry-aware representations of molecules provide a meaningful advantage over classical descriptor-based baselines on a pharmaceutical prediction task?} The task was originally drug-target binding for a single protein. We refined it to drug toxicity because the Therapeutics Data Commons benchmarks~\cite{huang2021} provide mature toxicity datasets with standardised splits and the methodological question transfers cleanly. DTI on DAVIS and KIBA is already loaded in the pipeline and is the planned target of the final phase (\S\ref{sec:nextsteps}). This report is a mid-term checkpoint: it documents the exploration, defines the problem precisely, and motivates the final-phase plan.

Going in, we expect graph representations to preserve more relevant structural information than fixed-length fingerprints, particularly around ring systems, hydrogen bonding, and stereochemistry. Classical fingerprint baselines should be hard to beat on small datasets and on highly imbalanced tasks, where flexible models risk overfitting the minority class. Whether a GNN advantage emerges on DTI, where the protein cannot be reduced to a fingerprint, is the open question.

\subsection{Baseline hierarchy}

Following the structure used in our own \texttt{baselines\_ames.ipynb}, the project is organised as a five-level hierarchy. Each level is meant to answer one specific question, and each level must beat the previous one to justify itself.

\section{Datasets and Evaluation}

\subsection{Datasets}

We use six toxicity datasets from Therapeutics Data Commons (TDC)~\cite{huang2021}. Tox21 is treated as twelve separate single-task endpoints, giving 17 tasks in total (4 binary classification, 1 regression, 12 Tox21 endpoints). Two drug-target interaction datasets (DAVIS and KIBA) are loaded into the same pipeline but reserved for the final leg.

The collection deliberately spans size (475 to 13{,}445 molecules), class balance (4\% to 50\% positive), and task type. ClinTox and the lower-prevalence Tox21 endpoints provide the imbalance stress test, AMES and hERG a balanced reference, DILI a small-data corner case, and LD50 the regression task that strains representation quality.

\subsection{Splits and metrics}

Classical baselines and the imbalance experiments use scaffold-based train, validation, and test splits, which group molecules by their core chemical skeleton so that test molecules belong to scaffold families unseen during training. This is the standard generalisation test for molecular property prediction. The GNN runs to date use TDC's deterministic random split, and reconciling both pipelines on scaffold splits is the first item in the final-leg plan.

We report ROC-AUC as the primary classification metric (threshold-independent, robust to imbalance), F1 of the positive class as a secondary metric (it reveals whether the model commits to predictions on the minority class), and $R^2$ and RMSE for LD50 regression. Accuracy is intentionally excluded from comparison: at 4\% prevalence on SR-ATAD5, predicting ``non-toxic'' for everything is 96\% accurate but useless. Where decision thresholds are used (for F1), we report both the default 0.5 cut-off and a calibrated threshold equal to the dataset's positive-class ratio.

\section{Methods}

\subsection{Molecular representations}

Four input representations are compared. \textbf{Morgan fingerprints (ECFP4)} encode each molecule as a 1024-bit vector via RDKit with radius~2~\cite{morgan1965, chemrev}, where each bit flags a circular substructure pattern. This is the QSAR default and the baseline any new method must beat. \textbf{RDKit 2D descriptors} are a small set of hand-engineered global properties (molecular weight, LogP, TPSA, hydrogen-bond donor and acceptor counts) that medicinal chemists routinely inspect but that lose all sub-structural detail. \textbf{ChemBERTa} is a RoBERTa backbone pre-trained on 77 million SMILES strings, used here with the backbone frozen and a small classification head trained on the toxicity label, to test whether large-scale unsupervised pre-training transfers downstream. \textbf{Molecular graphs} are produced by parsing each SMILES into a PyTorch Geometric \texttt{Data} object via RDKit~\cite{fey2019}, with atoms as nodes and bonds as edges in both directions. Atom features are described per GNN configuration in \S\ref{sec:gnn}, and bond-level features are not yet encoded (the edge index carries connectivity only).

\subsection{Classical baselines}

Random Forest (scikit-learn, 500 trees, \texttt{min\_samples\_leaf}=2) and XGBoost (500 estimators, depth 6, learning rate 0.1) are trained on Morgan fingerprints for every classification dataset, on the same input for LD50 regression, and additionally on RDKit 2D descriptors on AMES for the representation comparison.

\subsection{GNN architectures}
\label{sec:gnn}

All four architectures share an identical skeleton: two convolutional layers, ReLU activation, global pooling over each graph, and a linear output head. Any performance difference is therefore attributable to the aggregation rule. References: GCN~\cite{kipf2017}, GIN~\cite{xu2019}, GAT~\cite{velickovic2018}, GraphSAGE~\cite{hamilton2017}.

\begin{table}[H]
\centering
\caption{GNN architectures compared.}
\label{tab:archs}
\begin{tabular}{llll}
\toprule
\textbf{Model} & \textbf{Aggregation} & \textbf{Pooling} & \textbf{Theoretical property} \\
\midrule
GCN       & degree-normalised mean              & mean & symmetric Laplacian smoothing \\
GIN       & learnable $(1{+}\varepsilon)$-sum + MLP & sum  & as expressive as 1-WL test \\
GAT       & multi-head attention-weighted mean  & mean & learns per-edge importance \\
GraphSAGE & concat(self, mean(neighbours))      & mean & inductive generalisation \\
\bottomrule
\end{tabular}
\end{table}

We trained each architecture in two configurations. \textbf{v1} uses a minimal 4-feature atom encoding (atomic number, degree, formal charge, aromaticity), hidden dimension 16, learning rate 0.01, and unweighted binary cross-entropy. \textbf{v2} expands the atom encoding to 9 features (adding hydrogen count, ring membership, hybridisation, total valence, and radical electrons), increases hidden dimension to 64, lowers the learning rate to 0.001, and replaces the loss with class-weighted BCE: $\texttt{pos\_weight} = n_{\text{neg}} / n_{\text{pos}}$ computed per dataset from the training split.

\section{Results}

\subsection{Classical baselines (Level 1)}
\label{sec:cls_baselines}

Random Forest and XGBoost on Morgan fingerprints set the reference performance any learned representation must beat.

\begin{table}[H]
\centering
\caption{Classical ML baselines on Morgan fingerprints (scaffold splits).}
\label{tab:baselines}
\begin{tabular}{lrr}
\toprule
\textbf{Dataset} & \textbf{RF} & \textbf{XGBoost} \\
\midrule
AMES (50\% pos)            & 0.906 & 0.881 \\
hERG\_Karim (40\% pos)     & 0.920 & 0.897 \\
DILI (45\% pos)            & 0.868 & n/a \\
ClinTox (8\% pos)          & 0.792 & 0.870 \\
Tox21 (macro, 3 to 15\% pos) & 0.776 & 0.803 \\
\midrule
LD50\_Zhu ($R^2$)          & 0.583 & 0.605 \\
\bottomrule
\end{tabular}
\end{table}

On balanced classification, fingerprints reach 0.88 to 0.92 AUROC. On severely imbalanced ClinTox, RF drops to 0.79. On LD50 regression, both methods plateau at $R^2 \approx 0.6$, leaving roughly 40\% of the variance in acute toxicity unexplained.

\subsection{Class-imbalance interventions}
\label{sec:imbalance}

Because over-prediction of the majority class is the most visible failure mode on toxicity data, we tested two standard interventions before moving to GNNs.

\textbf{Cost-proportionate rejection sampling.} A custom meta-estimator computes a per-batch positive-class weight and probabilistically rejects majority-class samples. Under 5-fold cross-validation, mean ROC-AUC dropped to 0.794 on ClinTox and 0.744 on Tox21, a noticeable regression from the unmodified baselines. The cause is information loss: rejecting thousands of non-toxic molecules to match a tiny positive set throws away most of what the trees were learning from.

\textbf{SMOTE oversampling.} XGBoost with SMOTE-balanced training data, evaluated by per-class precision, recall, and F1 under 5-fold CV. Figure~\ref{fig:smote} shows the resulting confusion matrices. On Tox21 the model recovered useful minority-class behaviour (positive-class recall 0.46, F1 0.41). On ClinTox the gain was marginal (recall 0.20, F1 0.20). The asymmetry is informative. When fingerprints already separate the classes well (Tox21, large absolute number of positives), SMOTE helps the decision boundary. When fingerprints lack separating information (ClinTox, only $\sim$119 positives), no resampling can manufacture it.

\begin{figure}[H]
\centering
\includegraphics[width=0.92\textwidth]{smote_comparison.png}
\caption{Confusion matrices after SMOTE oversampling on the classical XGBoost pipeline. Tox21 (right) recovers a meaningful minority-class signal (125 of 272 toxic captured). ClinTox (left) does not (21 of 103). SMOTE cannot manufacture separating information that the input representation lacks.}
\label{fig:smote}
\end{figure}

These results establish the baseline lesson: \emph{imbalance is best addressed inside the loss function, not by mutilating the training set}. This directly motivated the class-weighted-loss choice for GNN~v2 (\S\ref{sec:gnn}).

\subsection{GNN results, v1 to v2 (Level 3)}

The four architectures were trained in two configurations across the 17 tasks (68 runs per configuration). v1 used a minimal 4-feature atom encoding, hidden dimension 16, unweighted BCE, and a fixed 0.5 threshold. v2 expanded atom features to 9 dimensions, raised hidden dimension to 64, and used class-weighted BCE with per-dataset \texttt{pos\_weight}.

\label{tab:v2}

v1 surfaced two failure modes. LD50 regression was under-fit across all four ($R^2 \le 0.43$), pointing at insufficient atom information and capacity. F1 collapsed on imbalanced classification: GCN and GAT produced F1~$=0$ at the default threshold despite reasonable AUROC. The clearest case was ClinTox/GIN, ROC-AUC 0.922 with F1~$=0$, where the model was ranking molecules near-perfectly but never crossing the sigmoid threshold. Calibrating the threshold to the class prior recovered F1 to 0.34 with no retraining.

v2 fixes both failures. Three of four architectures improve on every metric, and GIN remains strongest (Tox21 macro 0.781, 11 of 16 wins, LD50 $R^2 = 0.491$). The class-weighted loss is the largest single contributor to F1 recovery (GAT mean F1 jumps from 0.122 to 0.314). GraphSAGE regresses on three highly imbalanced datasets (ClinTox, NR-ER-LBD, NR-Aromatase) under aggressive \texttt{pos\_weight}, which focal loss~\cite{lin2017} addresses.

\section{Discussion}

\subsection{Where the fingerprint baseline is weak}

Across the toxicity stack, classical ML on Morgan fingerprints loses (or comes close to losing) only on small data (DILI, $n{=}475$) and on severe class imbalance (ClinTox 8\%). Where it wins comfortably (AMES, hERG, LD50), the dataset is large or balanced or both. The frozen ChemBERTa on AMES tightens the picture: a 77M-parameter chemical language model loses to a 1024-bit fingerprint plus 500 trees by 10 AUROC points. Pre-trained learned embeddings do not automatically transfer.

\subsection{Loss-side vs.\ data-side imbalance fixes}

Three independent results on the imbalance question converge:
\begin{itemize}
\item Rejection sampling on classical ML \emph{reduces} AUROC (information loss).
\item SMOTE on classical ML helps when fingerprints already separate the classes (Tox21) and is ineffective when they do not (ClinTox).
\item Class-weighted BCE inside the GNN loss roughly triples mean F1 on the architectures that had collapsed, with no data manipulation.
\end{itemize}
The lesson generalises beyond this project: imbalance interventions should reshape the gradient signal, not the dataset. The picture of v2 in Figure~\ref{fig:threshold} (right) is the picture of a model that learned to commit to the minority class during training, not one rescued at evaluation time by lowering a threshold.

\subsection{Limitations}

\begin{description}[leftmargin=1.5em, style=nextline]
\item[Inconsistent splits across pipelines.] Classical baselines and the SMOTE / rejection-sampling experiments use scaffold splits. The GNN runs use TDC's deterministic random split. The head-to-head table above is therefore an indication, not yet a fair comparison, and re-running the GNNs on scaffold splits is the first item in the next-steps plan.
\item[Single seed.] Each configuration was trained once. We have no confidence intervals. Differences smaller than roughly 2 AUROC points should not be over-interpreted.
\item[No bond features in the GNNs.] Bonds carry only connectivity, not type, stereochemistry, or ring membership at the edge level. This is the largest within-graph information gap.
\item[ChemBERTa tested on one dataset only.] We do not know whether learned embeddings fail under imbalance the same way the v1 GNNs did.
\item[No DTI experiments yet, no 3D yet.] Levels 4 (3D) and the dataset-axis extension (DTI) are not started.
\end{description}

\section{Final Phase}
\label{sec:nextsteps}

\subsection{Research question}

The proposal's question is answered narrowly on toxicity: GIN matches fingerprints only on small-data and severely imbalanced regimes, and trails on the rest. Pushing further along the representation axis (more features, 3D) would explore the same regime where the engineered baseline is already strong. The decisive test is drug-target interaction, where the protein partner cannot be Morgan-fingerprinted and the GNN drug encoder is paired with a protein encoder by construction. A fingerprint baseline must improvise on the protein side, while the GNN operates in its native regime. This is where the proposal originally aimed, and where the GNN's structural advantage should be largest.

\begin{quote}
\textit{Where does the graph neural network provide a meaningful advantage over Morgan-fingerprint baselines on drug safety prediction? On drug-only toxicity the answer is narrow: GIN matches fingerprints only on small or severely imbalanced regimes. On drug-target interaction, where the protein partner is not amenable to fingerprinting and a GNN drug encoder is paired with a protein encoder by design, does the GNN's structural advantage become decisive?}
\end{quote}

The mid-term answers the first half. The final phase answers the second.

\subsection{Plan}

The final phase hardens the GNN we already trained and takes it to drug-target interaction. The two steps are sequential. The DTI step is the one that answers the research question.

\begin{enumerate}[leftmargin=2em]
\item \textbf{Harden the GNN on toxicity.} Three changes targeting known weaknesses of v2. First, add bond-level edge features (type, stereochemistry, ring membership), the largest remaining within-graph information gap. Second, re-run all GNN configurations on scaffold splits with three seeds to give Table~\ref{tab:headtohead} confidence intervals. Third, replace GraphSAGE's aggressive \texttt{pos\_weight} with focal loss~\cite{lin2017} to recover its three regressed datasets. Output: a rigorous, scaffold-split, multi-seed head-to-head against the classical baseline on the same toxicity tasks.
\item \textbf{Apply the hardened GNN to DAVIS.} DAVIS is already loaded (30k drug-target pairs, 68 drugs $\times$ 442 kinase targets, $K_d$ regression). The GIN drug encoder from step~1 is paired with a frozen ESM-2 small protein encoder and a linear projection head on the target side, concatenated and passed through an MLP for the final scalar. The fingerprint baseline gets a Morgan fingerprint on the drug side and an amino-acid-composition embedding on the protein side. Metrics: Pearson, RMSE, concordance index. The win condition is whether the GNN drug encoder beats the fingerprint drug encoder when both are paired with the same protein representation, and by how much.
\item \textbf{Final analysis and report.} Combined comparison across toxicity and DTI on consistent splits with confidence intervals.
\end{enumerate}

\subsection{Stretch goals}

These run only if the DTI experiment converges with time remaining. They do not hold up the final report.

\begin{description}[leftmargin=1.5em, style=nextline]
\item[Level 4 (SchNet on LD50).] Train SchNet~\cite{schnet} on RDKit-generated 3D conformers for LD50. The cleanest place to test whether 3D atomic geometry adds anything that 2D plus bond features did not.
\item[KIBA.] The larger DTI dataset (118k pairs). Attempted only if DAVIS is solved decisively.
\end{description}

\section*{Software}

The pipeline is built on standard open-source libraries: RDKit for SMILES parsing and atom or bond featurisation, PyTorch and PyTorch Geometric~\cite{fey2019} for the GNNs, scikit-learn and XGBoost for the classical baselines, HuggingFace Transformers for ChemBERTa, and TDC~\cite{huang2021} for dataset access and scaffold splits. All experiments use seed 42 across \texttt{random}, \texttt{numpy}, \texttt{torch}, and CUDA. Per-version model weights, prediction CSVs, and TensorBoard logs are saved under \texttt{models/v\{1,2\}/} and \texttt{results/v\{1,2\}/} so each phase remains reproducible alongside later experiments.
